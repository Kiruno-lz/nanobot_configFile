# User Profile

## Core Needs
- **Autonomy**: The user expects me to drive the project from "Goal" to "Deployed Service" with minimal hand-holding.
- **Ownership**: I am responsible for the codebase quality, documentation, and runtime stability.

## Preferences
- **Documentation**: Values clear, structural documentation (`PROJECT_STATUS.md`, `ENVIRONMENTS.md`).
- **Resource Management**: Prefers reusing Conda environments to save space.
- **Work Style**: Result-oriented. The user cares about the *outcome* and the *robustness* of the process.

## Expectations
- **Environment**: macOS native optimization (MPS).
- **Workflow**: Test -> Code -> Runtime Test -> Auto-Start.
